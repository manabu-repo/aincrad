# 前端工程化

## 总览

> 前端工程化是一种通过使用工程化技术和工具来提高前端开发效率、优化项目结构、规范代码质量、降低维护成本的方法。

- Frontend Engineering: 前端工程

> Frontend engineering focuses on the engineering practices and tools used to build, maintain, and scale frontend web applications.

前端工程专注于用于构建、维护和扩展前端 Web 应用程序的工程实践和工具。

- Build Tools: 构建工具

> Build tools automate the process of compiling, bundling, and optimizing frontend code, making it more efficient and scalable.

构建工具可以自动执行编译、捆绑和优化前端代码的过程，使其更加高效和可扩展。

- Task Runners: 任务运行器

> Task runners automate repetitive tasks in the development workflow, such as compiling Sass, minifying JavaScript, and optimizing images.

任务运行器自动执行开发工作流程中的重复任务，例如编译 Sass、缩小 JavaScript 和优化图像。

- Module Bundlers: 模块打包工具

> Module bundlers combine separate modules or files into a single bundle that can be served to the browser, optimizing load times and reducing HTTP requests.

模块捆绑器将单独的模块或文件组合成一个可以提供给浏览器的捆绑包，从而优化加载时间并减少 HTTP 请求。

- Package Managers: 包管理工具

> Package managers streamline the process of managing dependencies and installing third-party libraries and frameworks.

包管理器简化了管理依赖项以及安装第三方库和框架的过程。

- Code Linters: 代码检查工具

> Code linters analyze code for potential errors, style violations, and best practices adherence, helping maintain code quality and consistency.

代码检查器分析代码中的潜在错误、风格违规和最佳实践遵守情况，帮助保持代码质量和一致性。

- Code Formatters: 代码格式化工具

> Code formatters automatically format code according to predefined rules or style guides, ensuring consistent code style across the project.

代码格式化程序根据预定义的规则或风格指南自动格式化代码，确保整个项目的代码风格一致。

- Version Control Systems (VCS): 版本控制系统

> Version control systems track changes to code, allowing developers to collaborate, revert changes, and maintain a history of project revisions.

版本控制系统跟踪代码更改，允许开发人员协作、恢复更改并维护项目修订历史记录。

- Continuous Integration (CI): 持续集成

> Continuous integration automates the process of integrating code changes into a shared repository, running tests, and providing feedback to developers.

持续集成自动执行将代码更改集成到共享存储库、运行测试并向开发人员提供反馈的过程。

- Continuous Deployment (CD): 持续部署

> Continuous deployment automates the process of deploying code changes to production environments, ensuring rapid and frequent releases.

持续部署自动化了将代码更改部署到生产环境的过程，确保快速、频繁的发布。

- Code Reviews: 代码审查

> Code reviews involve the systematic examination of code by peers or team members to identify bugs, improve code quality, and share knowledge.

代码审查涉及同行或团队成员对代码进行系统检查，以识别错误、提高代码质量和共享知识。

- Documentation Tools: 文档工具

> Documentation tools simplify the process of writing, generating, and maintaining project documentation, ensuring clarity and accessibility for developers.

文档工具简化了编写、生成和维护项目文档的过程，确保开发人员的清晰度和可访问性。
